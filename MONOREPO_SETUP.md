# Монорепо структура для React + Nest.js

## Предлагаемая структура:

```
react-grid-table/
├── frontend/              # React приложение (текущий проект)
│   ├── src/
│   ├── package.json
│   ├── vite.config.ts
│   └── ...
├── backend/               # Nest.js API
│   ├── src/
│   │   ├── main.ts
│   │   ├── app.module.ts
│   │   ├── tasks/         # Модуль задач
│   │   ├── database/      # Конфигурация БД
│   │   └── ...
│   ├── package.json
│   ├── nest-cli.json
│   └── tsconfig.json
├── shared/                # Общие типы и утилиты
│   ├── types/
│   │   └── jira.ts       # Общие TypeScript типы
│   └── package.json
├── package.json           # Root package.json для управления
└── README.md
```

## Преимущества монорепо:

✅ **Общие типы** - TypeScript типы можно использовать и на фронте, и на бэке
✅ **Единая разработка** - один `npm install` и `npm run dev` запускает всё
✅ **Проще тестировать** - всё в одном месте
✅ **Лучше для обучения** - видно связи между фронтендом и бэкендом

## Альтернатива: Отдельные репозитории

Если выберете отдельные репозитории:
- ✅ Независимое деплоирование
- ✅ Разные команды могут работать отдельно
- ❌ Сложнее синхронизировать типы
- ❌ Нужно настраивать CORS
- ❌ Больше репозиториев для управления

## Рекомендация

Для образовательного проекта - **монорепо** будет идеальным выбором!
